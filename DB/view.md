## view

데이터베이스에 존재하는 가상의 테이블을 의미한다.
실제로 행과 열이 존재하지만 데이터를 가지고 있는 것은 아니다.
다른 테이블이나 다른 뷰에 존재하는 데이터를 보여주는 역할만 수행 한다.

#### 필요성

- 사용자마다 특정 객체만 조회할 있음
    -  모든 직원에 대한 정보를 모든 사원이 볼 수 있도록 하면 안 됨
-  복잡한 질의문을 단순화 할 수 있음
- 데이터의 중복을 최소화할 수 있음

#### 생성방법

```sql
CREATE VIEW 뷰이름
SELECT name
FROM Customer
```

#### 장점

- 논리적 독립성을 제공함
- 데이터 접근 제어
- 사용자의 데이터 관리 단순화
- 여러 사용자의 다양한 데이터 요구 지원

#### 단점

- 뷰의 정의 변경 불가
- 삽입, 샂제, 갱신 영산이 제한이 있음
